# 🚀 Render Deployment Guide

## Quick Deploy to Render

### Prerequisites
- GitHub repository pushed to `GEMINI_API_PROD_READY` branch
- Render account (free tier available)
- Google Gemini API key

### Option 1: One-Click Deploy (Blueprint)

1. **Push to GitHub:**
```bash
git push origin GEMINI_API_PROD_READY
```

2. **Deploy via Render Dashboard:**
   - Go to [Render Dashboard](https://dashboard.render.com/)
   - Click "New" → "Blueprint"
   - Connect your GitHub repository
   - Select branch: `GEMINI_API_PROD_READY`
   - Render will auto-detect `render.yaml`
   - Click "Apply"

3. **Set Environment Variables:**
   - `GEMINI_API_KEY`: Your Google Gemini API key
   - `MASTER_API_KEY`: Your master API key for admin access
   - `SECRET_KEY`: Auto-generated by Render
   - `FORCE_HTTPS`: `true` (already set)

### Option 2: Manual Deploy

#### Step 1: Create Redis Instance
```bash
# Via Render Dashboard
1. New → Redis
2. Name: ai-ticket-classifier-redis
3. Region: Frankfurt (or closest to you)
4. Plan: Starter (Free)
5. Create Redis
```

#### Step 2: Create Web Service
```bash
# Via Render Dashboard
1. New → Web Service
2. Connect GitHub repository
3. Branch: GEMINI_API_PROD_READY
4. Name: ai-ticket-classifier-api
5. Region: Frankfurt
6. Runtime: Python 3
7. Build Command: pip install -r requirements.txt
8. Start Command: gunicorn --bind 0.0.0.0:$PORT --workers 4 --timeout 120 app:app
9. Plan: Starter ($7/month) or Free
```

#### Step 3: Configure Environment Variables
Add these in Render Dashboard → Service → Environment:

```bash
PYTHON_VERSION=3.12.0
FLASK_ENV=production
FORCE_HTTPS=true
GEMINI_API_KEY=<your-gemini-key>
MASTER_API_KEY=<your-master-key>
SECRET_KEY=<auto-generated>
REDIS_URL=<from-redis-service>
CORS_ORIGINS=*
```

#### Step 4: Set Health Check
- Path: `/api/v1/health`
- This ensures Render monitors your service

### Verify Deployment

Once deployed, test your endpoints:

```bash
# Get your Render URL (e.g., https://ai-ticket-classifier-api.onrender.com)
export RENDER_URL="https://your-app.onrender.com"

# Health check
curl $RENDER_URL/api/v1/health

# API docs
open $RENDER_URL/api-docs

# Register API key
curl -X POST $RENDER_URL/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -H "X-Forwarded-Proto: https" \
  -d '{
    "email": "test@example.com",
    "name": "Test User",
    "organization": "Test Org"
  }'

# Classify ticket
curl -X POST $RENDER_URL/api/v1/classify \
  -H "Content-Type: application/json" \
  -H "X-API-Key: <your-api-key>" \
  -H "X-Forwarded-Proto: https" \
  -d '{"ticket": "VPN connection keeps dropping"}'
```

### Troubleshooting

#### Service won't start
- Check logs: `Dashboard → Service → Logs`
- Verify Python version: 3.12
- Check environment variables are set

#### 403 Forbidden errors
- Ensure `X-Forwarded-Proto: https` header is sent
- Verify `FORCE_HTTPS=true` is set

#### Redis connection errors
- Verify `REDIS_URL` is correctly set from Redis service
- Check Redis service is running

### Custom Domain (Optional)

1. **Add Custom Domain:**
   - Dashboard → Service → Settings → Custom Domains
   - Add your domain (e.g., `api.yourdomain.com`)

2. **Configure DNS:**
   - Add CNAME record pointing to Render URL
   - Wait for SSL certificate provisioning
